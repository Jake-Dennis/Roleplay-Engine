--- ROOT CAUSE ANALYSIS ---
The scene-extraction.ts module uses the wrong column name:
  Line 32: "ORDER BY created_at DESC"
  Actual column: "timestamp" (verified via PRAGMA table_info)

This causes the SQL query to throw "no such column: created_at" error.
The error is caught by the outer try/catch (line 137-142) and logged as a warning.
The function returns without updating scene state — graceful failure, but feature is BROKEN.

Evidence:
  - messages table columns: id, session_id, sender_id, content, timestamp, location_context, emotional_tone, parent_message_id, is_deleted, deleted_at, persona_id
  - No "created_at" column exists
  - Query with "timestamp" works: returns 1 message
  - Query with "created_at" fails: "no such column: created_at"

VERDICT: FAIL — Integration broken due to wrong column name in scene-extraction.ts
